# 00_BOUSSOLE — par où commencer

Quatre documents à lire dans cet ordre quand on arrive sur le projet ou qu'on reprend après une coupure :

| # | Document | Pour quoi |
|---|---|---|
| 1 | [`ROADMAP-MAP.html`](../04_docs/11-roadmap-map.html) | Voir la carte des branches et d'où on part |
| 2 | [`CHANGELOG.md`](./CHANGELOG.md) | Voir ce qui a changé récemment |
| 3 | [`DECISIONS.md`](./DECISIONS.md) | Comprendre **pourquoi** les choses sont comme elles sont |
| 4 | [`GOUVERNANCE.md`](./GOUVERNANCE.md) | Les règles de conduite du projet |

## Règle d'or

Chaque session Cowork finit par :

1. Une entrée dans `CHANGELOG.md` (section `[Non publié]`)
2. Si décision ≥ 30 min débattue : une entrée dans `DECISIONS.md`
3. Commit git : `chore: session YYYY-MM-DD`

Sans cette hygiène, on perd la trace.
