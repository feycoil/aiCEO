01Vue d ensemble

aiCEO est votre Compass executive local pour CEO francophone CSP+. Il ne remplace pas vos Decisions, il les structure et les protege : un Cockpit le matin pour arbitrer ce qui compte (Triage), des automatismes pour digerer la masse d emails, un Bilan le soir pour clore proprement, un Weekly Sync pour ajuster.

Philosophie : 100 % local, SQLite mono-instance, zero cloud applicatif. Vos donnees restent sur votre poste. Anthropic Claude sert uniquement au raisonnement (chat assistant + 4 routes LLM frontend) - aucune donnee de votre Hub n est conservee chez Anthropic.

Les 4 cycles (Sync)

  • Triage matin (5-10 min) : tri emails + Decisions du jour
  • Cockpit journee : KPIs, Pulse, Top 3 du jour, North Star
  • Bilan soir (3 min) : mood, energie, top 3 fait, streak
  • Weekly Sync vendredi (30 min) : retrospective + Big Rocks + auto-draft Claude

06Triage du matin — le rituel en detail

Le Triage transforme votre boite mail en items actionnables en 5-10 minutes. C est la signature differenciante d aiCEO.

Les 4 verdicts possibles par email

  • Faire — cree un item dans la base selon le badge kind (action / decision / project / big-rock / info)
  • Reporter — garde l email actif pour la prochaine analyse
  • Ignorer — marque arbitre, pas de creation, pas d apprentissage
  • Pas pour moi (lien discret) — marque arbitre + envoie un signal a Claude pour qu il filtre les emails similaires a l avenir (table email_feedback, patterns dans le system prompt)

Les 5 kinds (cliquables sur le badge)

  • Action (violet) — tache operationnelle ponctuelle
  • Decision (ambre) — choix strategique a trancher (visible aussi en mode Focus decisions)
  • Project (vert) — sujet structurant. Si verdict Faire → auto-create projet (depuis S6.25.1) avec lien email source
  • Big Rock (orange) — promotion en rocher hebdomadaire (S6.25.2). Contrainte : 3 max par semaine ISO
  • Info (bleu) — contexte pour un projet existant, pas de creation task/decision (lie email a project_id)

Les deux modes

  • Mode Queue (emails) — bouton "Analyser avec Claude" lance une analyse LLM batch (12 emails → JSON enrichi summary/action/priority/rationale + skip noise). Bouton "Analyser (heuristique)" si pas de cle API.
  • Mode Focus (decisions) — cycle 1-by-1 sur les decisions actives + reportees + tranchees recentes (7 derniers jours, S6.26.4). 4 options : OUI / NON / Reporter / Plus d info.

Apprentissage actif (S6.24.2)

Une mini-card sur le Cockpit affiche en temps reel le nombre de patterns appris. Stats detaillees dans Reglages → Coaching IA : feedbacks par verdict, par mois, recents (10 derniers).

Reflexes UX

  • Bouton "Repondre" sur chaque card — ouvre Outlook avec un mailto pre-rempli (RE: subject)
  • Badge "✦ Claude" et bordure violette gauche sur cards analysees par LLM (S6.26.2)
  • Tag projet cliquable — reaffectation rapide d un email a un autre projet
  • Tour pedagogique au 1er Triage (S6.24.4) : 3 reflexes en overlay, dismiss permanent via localStorage
