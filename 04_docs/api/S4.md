## 1. Assistant chat (S4.01 — nouveau)

Module : `/api/assistant/*`. Stockage : tables `assistant_conversations` + `assistant_messages` (FK CASCADE). Migration : `data/migrations/2026-04-26-s4-assistant.sql`.

### `GET /api/assistant/conversations` — liste des 20 dernières conversations

```bash
curl -s http://localhost:4747/api/assistant/conversations | jq
```

```json
{
  "conversations": [
    {
      "id": "conv_xxx",
      "title": "Quelle est la capitale de la France ?",
      "created_at": "2026-04-26T14:32:11.000Z",
      "updated_at": "2026-04-26T14:32:14.000Z"
    }
  ],
  "count": 1
}
```

### `GET /api/assistant/conversations/:id` — historique complet

```bash
curl -s http://localhost:4747/api/assistant/conversations/conv_xxx | jq
```

```json
{
  "conversation": { "id": "conv_xxx", "title": "...", "created_at": "...", "updated_at": "..." },
  "messages": [
    { "id": "msg_1", "role": "user", "content": "Bonjour", "ts": "..." },
    { "id": "msg_2", "role": "assistant", "content": "Bonjour, comment puis-je aider ?", "ts": "...", "model": "claude-sonnet-4-6", "tokens_in": 12, "tokens_out": 28 }
  ]
}
```

### `POST /api/assistant/messages` — envoyer message + streaming SSE

```bash
curl -N -X POST http://localhost:4747/api/assistant/messages \
  -H "Content-Type: application/json" \
  -d '{"content":"Quels sont mes 3 chantiers prioritaires aujourd'\''hui ?"}'
```

Réponse : flux SSE (`text/event-stream`). Events :

```
event: conversation
data: {"id":"conv_yyy","title":"Quels sont mes 3 chantiers..."}

event: text
data: {"text":"Tes "}

event: text
data: {"text":"3 "}

event: text
data: {"text":"chantiers "}
…
event: done
data: {"tokens_in":42,"tokens_out":189,"model":"claude-sonnet-4-6"}
```

**Continuer une conversation existante** : ajouter `conversation_id` :

```bash
curl -N -X POST http://localhost:4747/api/assistant/messages \
  -H "Content-Type: application/json" \
  -d '{"content":"Et demain ?","conversation_id":"conv_yyy"}'
```

**Mode démo** (sans `ANTHROPIC_API_KEY` ou `DEMO_MODE=1`) : le serveur stream un message factice contenant `Mode fallback`. Permet tests offline + démo sans coût Claude.

### `DELETE /api/assistant/conversations/:id` — supprimer (cascade messages)

```bash
curl -X DELETE http://localhost:4747/api/assistant/conversations/conv_yyy
```

```json
{ "ok": true, "deleted": "conv_yyy" }
```

## 2. Groupes (S4.03 — page consommatrice)

Routes existantes utilisées par `groupes.html` :

```bash
# Liste groupes + counts projets
curl -s http://localhost:4747/api/groups | jq '.groups[] | {id, name, counts}'

# Détail groupe + projets attachés
curl -s http://localhost:4747/api/groups/grp_strategy | jq '.group.projects'
```

## 3. Projets (S4.04 + S4.05 — pages consommatrices)

```bash
# Liste tous les projets (filtres : group, status, q)
curl -s 'http://localhost:4747/api/projects?group=grp_strategy&status=hot' | jq

# Détail projet avec counts (tasks, decisions, contacts, events)
curl -s http://localhost:4747/api/projects/proj_amani-credit | jq

# Tâches d'un projet
curl -s 'http://localhost:4747/api/tasks?project=proj_amani-credit' | jq '.tasks | length'

# Décisions d'un projet
curl -s 'http://localhost:4747/api/decisions?project=proj_amani-credit' | jq '.decisions | length'
```

La page `/projet?id=xxx` aggrège ces 3 appels en parallèle (`Promise.all`).

## 4. Contacts (S4.06 — page consommatrice)

```bash
# Recherche full-text (name/email/company/role)
curl -s 'http://localhost:4747/api/contacts?q=Ansquer' | jq

# Filtre niveau de confiance
curl -s 'http://localhost:4747/api/contacts?trust_level=haute' | jq '.contacts[] | .name'
```

## 5. Décisions (S4.07 — page consommatrice + recommend IA)

```bash
# Liste filtrée
curl -s 'http://localhost:4747/api/decisions?status=ouverte' | jq

# Transitions de statut
curl -X POST http://localhost:4747/api/decisions/dec_xxx/decide
curl -X POST http://localhost:4747/api/decisions/dec_xxx/execute
curl -X POST http://localhost:4747/api/decisions/dec_xxx/abandon
```

**Recommandation IA** (S4.07) : la page `/decisions` envoie un prompt contextualisé à `POST /api/assistant/messages` (réutilise le streaming SSE existant), pas de nouvel endpoint dédié.
