# PARTIE 2 — PRINCIPES DESIGN

## Les 7 principes fondateurs aiCEO

### 1. Densité tenue, jamais saturée

**Pourquoi** : un CEO traite des dizaines d'informations à la seconde. Un dashboard qui crie épuise.
**Comment** : padding 32px, gouttière 24px, 3 niveaux hiérarchie max, couleur = sémantique pas décoration.
**Anti** : Salesforce, SAP, Power BI, Jira "data wall".

### 2. Couleur = sémantique, jamais décoration

**Pourquoi** : la couleur doit transporter de l'information, sinon elle pollue.
**Comment** : rose = alerte / urgence ; emerald = stable / done ; amber = attention / deadline proche ; sky = info ; violet = IA / coaching.
**Anti** : Trello, Asana, Monday qui colorent pour "égayer".

### 3. Anti-flatterie, factuel

**Pourquoi** : un CEO de 40 ans ne veut pas être traité comme un enfant.
**Comment** : "Arbitrage enregistré." pas "Bravo super !". "Erreur réseau." pas "Oups !".
**Anti** : DuoLingo, Headspace, productivity apps grand public.

### 4. Ergonomie cognitive : minimiser les décisions UI

**Pourquoi** : decision-fatigue chez le CEO. Chaque clic UI gratuit consomme de l'énergie.
**Comment** : defaults intelligents, action principale toujours évidente, raccourcis clavier, command palette.
**Anti** : interfaces avec 15 boutons "secondary" tous au même niveau visuel.

### 5. Coaching subtil, jamais infantilisant

**Pourquoi** : observer ≠ juger. Le copilot accompagne, ne dirige pas.
**Comment** : "J'ai noté que…" plutôt que "Tu devrais…". Questions ouvertes plutôt que conseils.
**Anti** : apps coach qui balancent des injonctions ("Buvez de l'eau !").

### 6. Performance native > framework lourd

**Pourquoi** : un outil utilisé 8h/jour doit être instantané.
**Comment** : vanilla JS où c'est possible, code split, critical CSS inline, service worker, SVG sprite.
**Anti** : sites en React qui mettent 5s à charger.

### 7. Accessibilité par défaut, pas en option

**Pourquoi** : WCAG AAA = qualité globale, pas "fonctionnalité accessibilité".
**Comment** : tab order naturel, focus visible, ARIA roles, contrastes vérifiés, tested with screen reader.
**Anti** : "on fera l'accessibilité plus tard" → impossible à rattraper.

## Voice & tone guidelines

### Voice

- **Direct** : pas de circonlocutions. Sujet + verbe + complément.
- **Factuel** : observation, pas opinion. "Outlook synchronisé il y a 4h" pas "Outlook a peut-être un problème".
- **Concis** : <60 chars titres, <140 chars corps, <30 chars chips.
- **Bienveillant** : recovery > culpabilisation. "Pas grave" pas "Dommage".
- **Sans flatterie** : jamais "Bravo", "Génial", "Super".

### Tone par contexte

| Contexte | Adjectifs |
|---|---|
| Confirmation action | sobre, factuel |
| Erreur réseau | factuel, action claire |
| Empty state | court, suggérer |
| Coaching weekly | question ouverte, factuel |
| Recovery (streak break) | bienveillant, sans culpabilisation |
| Friction positive | doux, interrogatif |
| Mirror moment | observation neutre, pas de jugement |
| Onboarding | clair, pas surchargé |

# PARTIE 3 — MESURE D'IMPACT CEO

## Score "santé exécutive"

### Composition

Score 0-100, calculé hebdomadairement, affiché dans la revue dimanche.

| Composante | Pondération | Source |
|---|---|---|
| **Régularité boucle du soir** | 20 % | Streak / 7 derniers jours |
| **Energie moyenne** | 15 % | Avg `energy` (1-5) sur 7j |
| **Mood moyen** | 10 % | Avg `mood` (mauvais=1, moyen=2, bien=3) sur 7j |
| **Tasks done / planned ratio** | 15 % | (tasks done) / (tasks planned) |
| **Decisions tranchées / reportées ratio** | 10 % | (decided) / (decided + deferred) |
| **Big Rocks atteints** | 15 % | (big rocks done) / (big rocks set) |
| **Time saved IA** | 10 % | Min économisées par drafts + délégations |
| **Décision velocity** | 5 % | Délai moyen entre création décision et tranchée |

### Affichage

```html
<div class="health-score">
  <svg class="health-ring" viewBox="0 0 120 120">
    <circle cx="60" cy="60" r="50" stroke="var(--color-surface-3)" stroke-width="10" fill="none"/>
    <circle cx="60" cy="60" r="50" stroke="var(--color-emerald)" stroke-width="10" fill="none"
            stroke-dasharray="314.16" stroke-dashoffset="94.25"
            stroke-linecap="round" transform="rotate(-90 60 60)"/>
    <text x="60" y="68" text-anchor="middle" font-size="32" font-weight="700">70</text>
  </svg>
  <div class="health-meta">
    <div class="health-label">Santé exécutive</div>
    <div class="health-trend">↗ +5 vs S16</div>
  </div>
</div>
```

### Politique

- Score affiché **uniquement** dans revues hebdo (dimanche)
- Pas de notification si score baisse
- Tendance affichée mais sans drame

## Time saved metric

### Calcul

```
time_saved_weekly =
  Σ (drafts générés × 8 min)
  + Σ (recommandations IA acceptées × 5 min)
  + Σ (délégations 1-clic × 3 min)
```

### Affichage

Footer cockpit : "Cette semaine : 3h12 économisées grâce au copilote."

Détail expandable :
- 18 drafts × 8 min = 2h24
- 12 recos × 5 min = 1h00 (mais 6 acceptées seulement = 30 min)
- 9 délégations × 3 min = 27 min

## Decision velocity

### Calcul

`avg(decision.decided_at - decision.created_at)` sur les 30 derniers jours.

### Affichage

Mirror moment dimanche : "Tu tranches en {time} en moyenne (vs {previous} il y a 3 mois)."

## Strategic vs tactical ratio

### Calcul

Tâches taggées `strategic=true` / total tâches done.

Tag automatique :
- `strategic=true` si la tâche est liée à un Big Rock OU si elle nécessite > 60min OU si type=`plan`
- `tactical` sinon

### Cible

70% strategic / 30% tactical (objectif de vie pro CEO).

### Affichage

KPI cockpit (compteurs) : "Stratégique : 55% (cible 70%)"

## Quarterly review (V1+)

Rapport trimestriel auto-généré :
- Health score moyen Q1 vs Q4
- Top 3 victoires
- Top 3 regrets
- Big Rocks atteints / définis
- Décisions tranchées
- Recommandations Claude pour Q+1

## Politique éthique

⚠ **Important** : ces métriques ne doivent **jamais** :
- Être affichées de manière agressive
- Servir de base à des notifications "Tu es en retard sur ton score"
- Comparer entre utilisateurs (jamais de leaderboard)
- Pénaliser une mauvaise semaine

Elles servent au CEO pour **s'observer**, pas pour être jugé.
