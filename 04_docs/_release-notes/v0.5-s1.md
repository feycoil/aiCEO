# Backend stable SQLite — premier sprint v0.5 livré

**Version** : `v0.5-s1` · **Date** : 25 avr. 2026
**GitHub** : milestone=`v0.5-s1` · tag=`v0.5-s1` · commit=`141ca0f`

Sprint S1 du plan v0.5 livré en binôme CEO + Claude (sans dev externe). 14 tables, 41 routes REST, 23/23 tests verts. Hotfix *node:sqlite* en cours de sprint pour franchir le mur des prebuilds Node 24 sur Windows. Tag `v0.5-s1` posé sur `141ca0f`. Reste à passer la recette manuelle CEO côté Windows.

## Nouveautés

- **Schéma SQLite** — 14 tables métier + 2 utilitaires (`settings`, `schema_migrations`) + 18 indexes via 1 migration idempotente
- **41 routes REST** sur 6 modules : `/api/{tasks,decisions,contacts,projects,groups,events}` (CRUD complet + transitions + recherches)
- **Event sourcing** : toute mutation tâche logguée dans `task_events`
- **UUIDv7-like** pour les IDs (chronologiquement triables)
- Adaptateur `src/db-driver.js` sur `node:sqlite` — API compatible better-sqlite3 (drop-in)

## Améliorations

- Migration `data.js` → SQLite : 3 groupes / 14 projets / 28 tâches / 10 décisions / 25 contacts / 25 events (idempotente)
- Migration `data/*.json` → tables historiques (`arbitrage_sessions`, `evening_sessions`, `delegations`)
- Tests d'intégration via `AICEO_DB_OVERRIDE` : la base de prod n'est plus jamais touchée par `npm test`

## Correctifs

- **S1.13 — Hotfix node:sqlite** : pivot depuis `better-sqlite3@11` qui n'a pas de prebuilds Node 24 et qui échouait à compiler sans VS Build Tools sur le poste CEO. Zéro binaire natif désormais.

## Performance

- `npm install` : 5 dépendances pures JS, plus aucun build natif
