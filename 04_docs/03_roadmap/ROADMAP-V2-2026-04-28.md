## 2.5 Garde-fous validation (post-analyse stratégique 28/04 PM late)

**3 points de contrôle obligatoires** (cf. ADR v9) :

### J+1 — SPIKE de validation ADD-AI (0.5 j)
- Mesure baseline S6.8 vs ADD-AI Lean sur 1 feature équivalente
- Critères : vélocité ×1.2 / coût −20% / erreurs −30%
- **Si NO-GO → abandon ADD-AI**, retour méthode classique
- Doc : `04_docs/_sprints/SPIKE-VALIDATION-ADD-AI/`

### Semaine 4 — Sprint S6.16 BETA Lamiae (1 j + 3 j calendaire)
- Setup poste Lamiae + 3 j d'usage autonome + debrief
- Critères : NPS ≥ 6/10, time-to-first-decision < 5 min
- **Si NO-GO → ajustement majeur roadmap** avant Phase 2
- Doc : `04_docs/_sprints/S6.16-beta-lamiae/`

### Semaine 12 — V1.0 Release
- Tous critères de sortie V1 validés
- Push origin + tag + GitHub Release
- Beta closed avec 5 CEO pairs francophones (V1.1)

## 4. Critères de sortie V1

À la fin de S7.11 :
- ✅ Note produit ≥ 9.0/10 (cf. AUDIT-UXUI grille)
- ✅ Smoke 100% (≥ 24/24)
- ✅ 7 KPIs produit instrumentés (cf. PLAN-REALIGNEMENT §2)
- ✅ Vélocité arbitrage ≥ 5 emails/min (mesurée)
- ✅ Onboarding < 5 min jusqu'à 1ère décision tranchée
- ✅ Coût LLM < 5€/CEO/mois (mesuré)
- ✅ NPS interne CEO + Lamiae ≥ 8/10
- ✅ 100% des pages migrées en v07 Atomic Templates
- ✅ Plugin Cowork `aiceo-dev` mature (10 skills + 8 subagents)
- ✅ Tag `v1.0` posé + GitHub Release

## 5. Risques V1 + mitigations

| Risque | Impact | Mitigation |
|---|---|---|
| Anthropic change pricing/tools | Élevé | Abstraction tool layer, possible bascule GPT/Mistral en 1 j |
| Lamiae rejette le produit en onboarding | Élevé | Sprint S7.9 dédié + interviews préalables + seed data généreux |
| Migration v06→v07 fait régresser visuellement | Moyen | Tests visuels diff vs maquette dans hooks pre-tag |
| Vélocité ADD-AI moins forte qu'espérée | Moyen | Retex mensuel pour ajuster méthode |
| Coût LLM hors budget | Faible | Batch S7.2 + cache + monitoring continu |
