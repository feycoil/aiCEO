## 0. Pour le CEO non-tech : comment lire ce document

> **Avant de plonger dans les chiffres**, voilà comment ce document est structuré et ce que les acronymes signifient :
>
> - Un **sprint** = un bloc de travail de 0.5 à 2 jours-binôme avec un objectif clair (ex: livrer une feature complète, ou faire un audit). Numéroté `S6.x`, `S7.x`.
> - Un **j-binôme** = une journée de travail entre Major Fey (CEO) et Claude (IA). Pas une journée de Claude seul.
> - **ADD-AI** (AI-Driven Development) = la méthode où c'est l'IA qui code, le CEO qui valide.
> - **Lean** = version minimaliste : on ne fait que ce qui est nécessaire pour valider la prochaine étape.
> - **Phase 0/1/2/3** = grandes étapes du projet, chacune produit un livrable mesurable.
> - **Garde-fou** = critère formel qui doit passer sinon on s'arrête (3 dans cette roadmap).
> - **V1.0** = version commercialisable du produit (cible : 12 semaines).
> - **Editorial Executive** = direction artistique adoptée le 28/04 PM (ADR v11). Hybride Reflect + Granola + Linear + Apple HIG. Tokens DS canoniques : ivory/ink warm neutrals, Crimson Pro serif (titres), Inter sans (corps), accent violet primary, 8-grid spacing, 2 niveaux d'élévation. Cf. [`DIRECTION-ARTISTIQUE.md`](../00_methode/DIRECTION-ARTISTIQUE.md).
>
> 📖 **Tous les autres acronymes sont définis dans le [Glossaire](../00_methode/GLOSSAIRE.md)** — accessible aussi via le pilotage.

## 5. Risques V1 + mitigations

| Risque | Impact | Mitigation |
|---|---|---|
| Anthropic change pricing/tools | Élevé | Abstraction tool layer, possible bascule GPT/Mistral en 1 j |
| Lamiae rejette le produit en onboarding | Élevé | Sprint S7.9 dédié + interviews préalables + seed data généreux |
| Migration v06→v07 fait régresser visuellement | Moyen | Tests visuels diff vs maquette dans hooks pre-tag |
| Migration Editorial Executive casse pages câblées (S6.4) | Moyen | S6.11-EE en sprint dédié + 1 page-pilote (decisions.html) validée avant rollout · alias compat legacy préservés (`--bg`, `--surface-*`, `--text*`, `--violet-*`...) · rollback Git par commit atomique |
| Vélocité ADD-AI moins forte qu'espérée | Moyen | Retex mensuel pour ajuster méthode |
| Coût LLM hors budget | Faible | Batch S7.2 + cache + monitoring continu |
